1 · Session state

v1.1 · near-term focus

At the table the numbers change — wounds, Void, spells. A local per-character overlay (Room) tracks it and never writes back to the .l5r; resettable to baseline.

  • ✅
    Room-backed session overlay
    Persistence layer for the whole theme; merged onto the derived baseline in AppViewModel, missing row = no overlay.
  • ✅
    Wounds tracker
    Combat tab: specific damage/heal entry, Rest & Reset; surfaces wound level + penalty via pure woundStatus().
  • ✅
    Void points
    Spend/regain against the pool (= Void Ring); one spend per round; refreshes on daily rest.
  • 🔜
    Spell slots
    Per-element counters (= element Ring) plus Void-Ring bonus slots; a failed cast still burns a slot.
  • 🔜
    Status conditions
    Toggle Blinded / Dazed / Fatigued / Prone / Stunned / Fear… and apply their roll & Armor-TN effects.
  • 🔜
    Session notes
    Free-form worklist for XP gained / loot / plot — deliberately not applied to the sheet; transcribe later on desktop.
  • 💭
    Active Kata / Kiho
    Track the one active Kata / Kiho and apply its modifiers; same overlay pattern.
  • 💭
    Monk tattoos (irezumi)
    Track which tattoo abilities are active and apply their effects.
  • 💭
    Shadowlands Taint
    Track the Taint rank as a session / persistent value.

2 · Table tools

v1.2 dice · v1.3 combat

Reduce the physical props at the table: a correct Roll & Keep engine and contextual rolls / combat tracking built on top of it.

  • ✅
    Roll & Keep engine
    XkY, exploding tens, Ten Dice Rule, Void boost, unskilled rolls, dice-penalty rule. Shipped v0.2.0.
  • ✅
    Stance selection
    Attack / Full Attack / Defence / Full Defence / Center — adjusts derived values and feeds rolls.
  • 🔨
    Contextual rolls from the sheet
    Tap a stat to roll the matching pool, pre-filled TN + wound penalty. Weapon attack/damage rolls have landed.
  • 🔨
    Combat tracker
    Round structure, live Armor TN, action economy, spell countdown. Initiative + stance have landed.
  • ✅
    Weapon selection
    Equip toggle in the Equipment section drives the attack skill + damage roll, surfaced in the Combat tab with stance-adjusted pools.
  • 🔨
    Raises & Free Raises
    Raise stepper: called Raises +5 TN (capped at Void), Free Raises off-cap / −5 TN. Engine supports raises.
  • 🔜
    Skill Mastery abilities
    Surface unlocked masteries (ranks 3/5/7) and auto-apply the ones touching rolls (Free Raises, flat bonuses).
  • ✅
    Armor selection
    Worn / removed toggle drops the armor's Armor TN + Reduction bonus, reflected live in the Combat tab (overlay, never rewrites the derived sheet).
  • 💭
    Maneuvers
    Raise-cost table (Disarm, Feint, Knockdown, Called Shot…) folded into the attack roll's Raises.
  • 💭
    Iaijutsu duel
    Guided 3-stage flow (Assessment → Focus → Strike) for the player's side.
  • 💭
    Grapple
    Track the grappled state and its per-round contested Strength check.

4 · Rules engine depth

exploratory

Beyond v1's flat bonuses: evaluate datapack ModifierDef expressions and the situational modifiers that never print on a static sheet. Two layers — static parity (A) vs play-time extension (B).

  • 💭
    Static-modifier parity (Layer A)
    Grow ModifierDef evaluation feeding the derived sheet; desktop stays source of truth.
  • 💭
    Desktop-oracle parity tests
    Golden tests across clans/schools/shugenja asserting derived numbers match the desktop.
  • 💭
    Modifier taxonomy
    Classify every modifier auto / conditional / manual — the data both layers key off.
  • 💭
    Graceful degradation
    Unevaluable modifiers become a toggle/prompt carrying the rule text — a checklist item, not a silent bug.
  • 💭
    Situational engine (Layer B)
    Conditional evaluator grown by trigger category (stance, weapon, target, once-per-X, Void…).

ThemeItemStatus

Design principle — two modifier layers

Layer A (derived sheet, static): fixed bonuses that print on a sheet — must match the desktop exactly, enforced by golden tests. Layer B (play-time, situational): conditional bonuses the desktop doesn't model — the app's own extension; contract is transparent & overridable, never silently wrong. Layer B is additive on top of A and never rewrites parity-tested numbers.

Considered but out of scope (for now)

Bidirectional QR (re-export from the app) · multi-device / GM↔player session sync — revisitable once session state lands.
